FAQ

Questions, answered

How do I add an image and then write under it?

Each section is a stack of blocks. Hit + Image to add a picture, then + Text to add another writing box below it — repeat as much as you like. You can also drag a file straight onto a section, or paste a screenshot from your clipboard with Ctrl/Cmd+V. Use the ↑ ↓ arrows on any block to reorder.

Where do the images actually show up?

In the Web page, Word and PDF exports, embedded directly in the file. Slack, email, Markdown and plain text are text-only formats, so images appear there as a labelled placeholder like [Image: staging error] — paste those screenshots in manually after posting.

Is my data sent anywhere?

No. There is no backend and no analytics, so your report and screenshots are never uploaded. Everything is saved only to this browser's local storage. The one network request the page can make is fetching the PDF-rendering library from a CDN, and only if you click PDF — the library comes down, your content does not go up.

Will lots of screenshots break it?

Images are automatically resized and compressed before they're stored, which keeps things fast. But browser local storage is capped at a few megabytes, so if you add a lot of large images the auto-save may not fit — you'll get a clear warning if that happens, and your report still exports perfectly. Just export before closing the tab.

What's the “% done” thing in In progress?

Optional. Write API refactor — 60% and the report renders a real progress bar for it in the web, Word and PDF exports. Skip it and the line stays a plain bullet.
